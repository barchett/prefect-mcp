<objective>
Create `src/parts.ts` containing Zod schemas for the OpenCode Part union (all 12 members) and supporting types (ToolState, ApiError). This module is the typed surface that downstream tasks (Plan 02 `opencode_run` SURF-02, Plan 03 `opencode_session_command` CMD-01) will import to validate and shape their responses. The schemas must mirror the SDK type definitions verbatim — discriminator strings, field names, and optionality.

Purpose: Avoid re-doing the v1 once/always/reject permission-enum class of bug. The Part union has 12 discriminator strings, ToolState has its own inner discriminator (`status`, NOT `type`), and SubtaskPart has no named SDK export. Getting any of these wrong silently breaks downstream parsing. Centralizing the schemas in one module means Plan 02 and Plan 03 share a single source of truth.

Output: A module exporting `PartSchema` (a discriminated union over 12 members) and the supporting schemas, plus a runtime parse test confirming live-shaped data parses without throwing.
