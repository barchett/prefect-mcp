<objective>
Create a `prefect init` CLI that writes a correctly-configured `.mcp.json` into the current working directory, with merge-not-overwrite semantics so it doesn't trample on a developer's existing MCP configuration. The CLI is the new bin entry for the `prefect` package — `package.json` `bin` is rewired from the MCP server entry point to the CLI entry point, and the CLI itself does NOT spawn the MCP server (it only writes config).

Per CONTEXT D-16 through D-18: entry point is `src/cli.ts` → `build/cli.js`. The `.mcp.json` template uses an absolute path to `build/index.js` resolved at runtime from the CLI's own location (so it works whether the CLI is run from a global install or a local clone). Merge behavior has four cases (D-17): create-fresh, add-prefect-key, refuse-without-force, and overwrite-with-force.

Purpose: A developer cloning Prefect can run `prefect init` once and have a working `.mcp.json` without hand-editing JSON. The merge semantics mean Prefect plays nicely alongside other MCP servers in the same project.

Output: A new `src/cli.ts` file, an updated `package.json` (`bin` field repointed, `build` script extended). No changes to `src/index.ts`. This plan is fully independent of Plan 02 and can run in parallel — file ownership is `src/cli.ts` (NEW) + `package.json` (different keys than Plan 03 touches: `bin` and `scripts`, not `dependencies`).
</objective>

Create `src/cli.ts` with the EXACT structure below. This file uses ESM imports (matches `"type": "module"`) and resolves the path to `build/index.js` at runtime so the same CLI works whether it's run from a global install (`/usr/local/lib/node_modules/prefect/build/cli.js`) or a local clone.

```typescript
#!/usr/bin/env node
import { readFileSync, writeFileSync, existsSync } from 'node:fs';
import { resolve, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

// Resolve absolute path to build/index.js (the MCP server) from this CLI's
// own location. Both files live side-by-side in the build/ output dir, so the
// MCP server is at the same directory as this CLI.
const __dirname = dirname(fileURLToPath(import.meta.url));
const mcpServerPath = resolve(__dirname, 'index.js');

// Template for the prefect entry written into .mcp.json mcpServers.prefect.
// Uses node + absolute path because relative paths break when Claude Code
// resolves the spawn command from a working directory other than the project root.
const PREFECT_ENTRY = {
  type: 'stdio',
  command: 'node',
  args: [mcpServerPath],
  env: {},
} as const;

function usageAndExit(): never {
  console.error('Usage: prefect init [--force]');
  process.exit(1);
}

const args = process.argv.slice(2);
const subcommand = args[0];
const force = args.includes('--force');

if (subcommand !== 'init') {
  usageAndExit();
}

const mcpJsonPath = resolve(process.cwd(), '.mcp.json');

type McpJson = {
  mcpServers?: Record<string, unknown>;
  [key: string]: unknown;
};

if (!existsSync(mcpJsonPath)) {
  // Case 1 (D-17): create fresh with only the prefect entry
  const config: McpJson = { mcpServers: { prefect: PREFECT_ENTRY } };
  writeFileSync(mcpJsonPath, JSON.stringify(config, null, 2) + '\n');
  console.error('Created .mcp.json with prefect entry');
  process.exit(0);
}

// Existing .mcp.json — parse, merge, write
let existing: McpJson;
try {
  existing = JSON.parse(readFileSync(mcpJsonPath, 'utf8')) as McpJson;
} catch (err) {
  console.error(`Error: failed to parse .mcp.json — ${(err as Error).message}`);
  process.exit(1);
}

const servers = (existing.mcpServers ?? {}) as Record<string, unknown>;

if ('prefect' in servers && !force) {
  // Case 3 (D-17): refuse without --force
  console.error('Error: .mcp.json already contains a prefect entry. Use --force to overwrite.');
  process.exit(1);
}

// Case 2 (no prefect key) or Case 4 (--force): set only the prefect key,
// preserving all other servers and root-level keys.
servers.prefect = PREFECT_ENTRY;
existing.mcpServers = servers;
writeFileSync(mcpJsonPath, JSON.stringify(existing, null, 2) + '\n');
console.error(force ? 'Updated prefect entry in .mcp.json' : 'Added prefect entry to .mcp.json');
process.exit(0);
```

Concrete details (do NOT deviate):
- `import.meta.url` + `fileURLToPath` is the canonical ESM way to get `__dirname` equivalent. Do NOT use `require.resolve` or `__dirname` (which is undefined in ESM).
- The entry shape MUST match the existing .mcp.json format (`type: 'stdio'`, `command`, `args`, `env`). Preserving `type: 'stdio'` keeps it compatible with Claude Code's MCP server discovery.
- `as const` on PREFECT_ENTRY: gives the tuple-typed args the most-specific type for TS inference; functionally equivalent to a regular object at runtime.
- `args: [mcpServerPath]` — the resolved ABSOLUTE path. Per D-18, no relative paths because Claude Code may spawn from any working directory.
- The merge logic spreads the existing root keys via `existing` directly (we only mutate the `mcpServers` sub-object, not the root). Root-level keys outside `mcpServers` are preserved automatically.
- `process.exit(0)` on success paths — explicit, even though Node would exit 0 anyway. Helps when the script is sourced or its exit code is awaited.
- All output is `console.error` — stderr only, matching the project convention. Non-error output (e.g. "Created .mcp.json...") still goes to stderr so the script is composable in pipelines that consume stdout.

Do NOT add: a Commander.js dependency (manual argv parsing is fine here per D-discretion), a separate `init.ts` module (one file is enough), interactive prompts (D-18 says no prompts), or any logic that touches `package.json`.

```typescript
import { test } from 'node:test';
import assert from 'node:assert/strict';
import { spawnSync } from 'node:child_process';
import { mkdtempSync, writeFileSync, readFileSync, existsSync, rmSync } from 'node:fs';
import { tmpdir } from 'node:os';
import { resolve, join } from 'node:path';

const CLI = resolve(process.cwd(), 'build/cli.js');

function freshTmp(): string {
  return mkdtempSync(join(tmpdir(), 'prefect-cli-'));
}

function runInit(cwd: string, ...args: string[]): { status: number; stderr: string } {
  const res = spawnSync('node', [CLI, ...args], { cwd, encoding: 'utf8' });
  return { status: res.status ?? -1, stderr: res.stderr };
}

test('Case 1: creates .mcp.json when none exists', () => {
  const dir = freshTmp();
  try {
    const { status } = runInit(dir, 'init');
    assert.equal(status, 0);
    const cfg = JSON.parse(readFileSync(join(dir, '.mcp.json'), 'utf8'));
    assert.ok(cfg.mcpServers.prefect);
    assert.equal(cfg.mcpServers.prefect.command, 'node');
    assert.equal(cfg.mcpServers.prefect.type, 'stdio');
    assert.ok(Array.isArray(cfg.mcpServers.prefect.args));
    assert.ok(cfg.mcpServers.prefect.args[0].endsWith('index.js'));
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});

test('Case 2: adds prefect entry, preserves siblings', () => {
  const dir = freshTmp();
  try {
    writeFileSync(join(dir, '.mcp.json'), JSON.stringify({
      mcpServers: { other: { command: 'sh', args: ['-c', 'echo hi'] } },
    }));
    const { status } = runInit(dir, 'init');
    assert.equal(status, 0);
    const cfg = JSON.parse(readFileSync(join(dir, '.mcp.json'), 'utf8'));
    assert.ok(cfg.mcpServers.prefect);
    assert.ok(cfg.mcpServers.other);
    assert.equal(cfg.mcpServers.other.command, 'sh');
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});

test('Case 3: exits 1 when prefect already present without --force', () => {
  const dir = freshTmp();
  try {
    writeFileSync(join(dir, '.mcp.json'), JSON.stringify({
      mcpServers: { prefect: { command: 'old', args: [] } },
    }));
    const { status, stderr } = runInit(dir, 'init');
    assert.equal(status, 1);
    assert.match(stderr, /--force/);
    // Verify .mcp.json untouched
    const cfg = JSON.parse(readFileSync(join(dir, '.mcp.json'), 'utf8'));
    assert.equal(cfg.mcpServers.prefect.command, 'old');
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});

test('Case 4: --force overwrites only the prefect key', () => {
  const dir = freshTmp();
  try {
    writeFileSync(join(dir, '.mcp.json'), JSON.stringify({
      mcpServers: {
        prefect: { command: 'old', args: [] },
        other: { command: 'sh' },
      },
    }));
    const { status } = runInit(dir, 'init', '--force');
    assert.equal(status, 0);
    const cfg = JSON.parse(readFileSync(join(dir, '.mcp.json'), 'utf8'));
    assert.equal(cfg.mcpServers.prefect.command, 'node');
    assert.equal(cfg.mcpServers.other.command, 'sh');
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});

test('Root-level non-mcpServers keys are preserved', () => {
  const dir = freshTmp();
  try {
    writeFileSync(join(dir, '.mcp.json'), JSON.stringify({
      theme: 'dark',
      mcpServers: { other: { command: 'sh' } },
    }));
    const { status } = runInit(dir, 'init');
    assert.equal(status, 0);
    const cfg = JSON.parse(readFileSync(join(dir, '.mcp.json'), 'utf8'));
    assert.equal(cfg.theme, 'dark');
    assert.ok(cfg.mcpServers.prefect);
    assert.ok(cfg.mcpServers.other);
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});

test('Bogus subcommand exits 1 with usage', () => {
  const dir = freshTmp();
  try {
    const { status, stderr } = runInit(dir, 'bogus');
    assert.equal(status, 1);
    assert.match(stderr, /Usage: prefect init/);
    assert.equal(existsSync(join(dir, '.mcp.json')), false);
  } finally {
    rmSync(dir, { recursive: true, force: true });
  }
});
```
