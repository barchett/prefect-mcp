    This task validates that the full Prefect loop works after setup. Run it once after a fresh setup to confirm `.mcp.json` is registered, OpenCode is reachable on `http://localhost:4096`, and the create -> run -> diff -> commit cycle produces a real diff in git history.

    Send this exact prompt to OpenCode via `opencode_run`:

    > Create a file at `examples/hello.ts` that exports a function `greet(name: string): string` which returns `'Hello, ' + name + '!'`. At the bottom of the file, add the line: `console.log(greet('World'));`

    This prompt is intentionally specified in full so the model has no ambiguity. The prompt must instruct OpenCode to write a file — Phase 1 UAT proved that prompts which only ask for a text reply return an empty diff.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project filesystem -> Claude Code subprocess | .mcp.json instructs Claude Code which binary to spawn; arbitrary `command` here would let an attacker who gains write access to the repo run code at session start |
| examples/test-task.md -> OpenCode runtime | The prompt in test-task.md is sent to OpenCode which has `bash:allow, edit:allow, write:allow` permissions per the local config; a malicious prompt could instruct destructive behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | T (Tampering) | .mcp.json | accept | Personal-use tool. The file references a relative path `build/index.js` inside this repo. If an attacker has write access to .mcp.json, they already have repo write access and can do worse. Trust boundary is the repo itself. |
| T-02-01-02 | E (Elevation of Privilege) | Subprocess spawn | accept | `command: "node"` resolves via PATH; we rely on the user having Node installed. If PATH is hijacked, the user already has bigger problems. Personal-use tool — security_enforcement is not enabled in config.json. |
| T-02-01-03 | I (Information Disclosure) | env field of .mcp.json | mitigate | `env: {}` is empty by design — no secrets land in the committed file. README documents that users can add OPENCODE_URL/PREFECT_TIMEOUT_MS overrides locally without committing. |
| T-02-01-04 | T (Tampering) | examples/test-task.md prompt | accept | Test prompt creates a single specific file (`examples/hello.ts`) inside the repo. Even if OpenCode misinterprets it, blast radius is bounded by the working tree (git is the safety net per PROJECT.md decision). |
</threat_model>
