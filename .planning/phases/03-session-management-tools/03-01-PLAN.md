```typescript
// SESSION-04: Retrieve message history for a session (limit = most-recent-N, no cursor)
server.registerTool(
  'opencode_session_messages',
  {
    description: 'Retrieve the message history for an OpenCode session. Each message includes an info object (UserMessage or AssistantMessage) and a parts array (TextPart, ToolPart, PatchPart, etc.). Use limit to cap the number of messages returned — this returns the most recent N messages only; there is no cursor or offset. Omit limit to return all messages.',
    inputSchema: z.object({
      sessionId: z.string().describe('Session ID'),
      limit: z.number().int().positive().optional().describe(
        'Maximum number of messages to return. Returns the most recent N messages — there is no offset or cursor. Omit to return all messages.'
      ),
      directory: z.string().optional().describe('Optional directory filter'),
    }),
  },
  async ({ sessionId, limit, directory }) => {
    try {
      const { data, error } = await client.session.messages({
        path: { id: sessionId },
        query: { ...(limit !== undefined ? { limit } : {}), ...(directory ? { directory } : {}) },
      });
      if (error) throw new Error(JSON.stringify(error));
      return { content: [{ type: 'text', text: JSON.stringify(data) }] };
    } catch (err) {
      return { content: [{ type: 'text', text: String(err) }], isError: true };
    }
  }
);
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP input → Zod schema | Tool arguments arrive from Claude Code over stdio JSON-RPC; Zod validates before handler runs |
| Zod-validated args → SDK path/query | sessionId and messageId are passed as path params to the OpenCode HTTP API — string values only |
| SDK → OpenCode HTTP server | OpenCode enforces further validation at the API layer; errors surface via { data, error } destructuring |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | sessionId path param in SESSION-02/04/05 | accept | String validated by Zod; OpenCode enforces existence at HTTP layer (404 on unknown ID); no injection risk — not used in shell or SQL |
| T-03-02 | Tampering | messageId path param in SESSION-05 | accept | Same as T-03-01; string validated by Zod; OpenCode 404s on unknown ID |
| T-03-03 | Information Disclosure | opencode_session_status returns all sessions | accept | Tool is scoped to a single local OpenCode instance accessed by the same developer; no multi-user surface |
| T-03-04 | Denial of Service | limit param in SESSION-04 could trigger large response | accept | OpenCode enforces its own resource limits; limit only reduces the response, never increases server load beyond a full fetch |
| T-03-05 | Elevation of Privilege | directory filter param on all tools | accept | directory is a query filter passed to OpenCode; OpenCode enforces path access; MCP server has no filesystem access itself |
</threat_model>
