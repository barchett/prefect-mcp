```typescript
// SESSION-06: Delete a session permanently (irreversible)
server.registerTool(
  'opencode_session_delete',
  {
    description: 'Delete an OpenCode session and all its data permanently. Returns true on success. WARNING: this is irreversible — all messages, parts, and session history will be deleted. Consider using opencode_session_rename to archive instead of deleting.',
    inputSchema: z.object({
      sessionId: z.string().describe('Session ID to delete'),
      directory: z.string().optional().describe('Optional directory filter'),
    }),
  },
  async ({ sessionId, directory }) => {
    try {
      const { data, error } = await client.session.delete({
        path: { id: sessionId },
        query: directory ? { directory } : undefined,
      });
      if (error) throw new Error(JSON.stringify(error));
      return { content: [{ type: 'text', text: JSON.stringify(data) }] };
    } catch (err) {
      return { content: [{ type: 'text', text: String(err) }], isError: true };
    }
  }
);
```

// SESSION-07: Rename a session — MCP tool is "rename" but SDK method is client.session.update()

      sessionId: z.string().describe('Session ID to rename'),
      title: z.string().describe('New display title for the session'),

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP input → Zod schema | Tool arguments arrive from Claude Code over stdio JSON-RPC; Zod validates before handler runs |
| Zod-validated sessionId → SDK path param | sessionId is passed as { id } in path — string only, no shell exposure |
| SDK → OpenCode HTTP server | OpenCode enforces session existence (400/404); errors surface via { data, error } |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | SESSION-06 delete — irreversible destruction | mitigate | Tool description explicitly warns "irreversible" and "all data deleted permanently"; suggests opencode_session_rename as alternative; caller must provide sessionId (validated by Zod as string); no accidental deletion without explicit call |
| T-03-07 | Tampering | SESSION-07 rename — body title field | accept | title is a free-form string passed through to OpenCode; OpenCode validates server-side; empty string behavior is server-determined (see Open Question in RESEARCH.md); no injection risk |
| T-03-08 | Tampering | SESSION-09 unrevert — modifies session revert state | accept | Only valid on reverted sessions (OpenCode 400s otherwise); calling on non-reverted session fails safely; no cascade risk |
| T-03-09 | Information Disclosure | SESSION-08 children — exposes session hierarchy | accept | Same single-developer local OpenCode instance scope as all other tools; no multi-user surface |
| T-03-10 | Denial of Service | SESSION-06 delete of active session | accept | OpenCode enforces session lifecycle rules; if session is busy, OpenCode may 400/409; MCP server surfaces error normally |
</threat_model>
