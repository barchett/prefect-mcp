# Phase 15.1 Plan 01: maxSessions Schema, CLI Flag, CAPACITY Column, and countSessionsForServer Summary

**One-liner:** maxSessions field on ServerEntry with --max-sessions CLI flag, CAPACITY table column, and countSessionsForServer() capacity counting primitive for Plan 02 enforcement.

## What Was Built

### Task 1: maxSessions field + countSessionsForServer helper
- Added `maxSessions?: number` to `ServerEntry` interface in `src/registry.ts` — optional, passes through JSON naturally without changes to addServer/writeRegistry/readRegistry
- Added CAPACITY column to `listServers` output: header shows `CAPACITY`, rows show the integer or `'unlimited'`; MODEL column padded to 17 chars (was unpadded last column)
- Added `countSessionsForServer(serverName, sessionsPath?)` export to `src/sessions.ts` — filters `readSessionMap().sessions` by `entry.server === serverName`, returns count; returns 0 on ENOENT
- Added 7 MULTI-11 tests to `src/registry.test.ts`: maxSessions round-trip, no-maxSessions omission from JSON, countSessionsForServer with matching/absent/missing-file sessions, CAPACITY column header and numeric display

### Task 2: --max-sessions CLI flag, capacity bullet, CLI tests
- Updated `handleAddServer` in `src/cli.ts` to extract `--max-sessions <n>` flag from any position in args, validate as positive integer (exits 1 on non-integer or zero), spread into addServer call only when present
- Updated `updateClaudemdWorkers` bullet format to append `, capacity: {n|unlimited}`
- Updated `usageAndExit` to show `[--max-sessions <n>]` in add-server usage line
- Added 8 MULTI-11 tests to `src/cli.test.ts`: flag stores integer, absent flag omits field, non-integer exits 1, zero exits 1, list-servers CAPACITY header, list-servers numeric capacity, CLAUDE.md bullet with capacity value, CLAUDE.md bullet with 'unlimited'
