<locked_decisions>
From .planning/STATE.md (NON-NEGOTIABLE):

1. **Uniform directory schema** — Add `directory` param to ALL 18 tools in the Zod schema. Pass it through to the SDK on every tool. All 18 SDK *Data types accept it without error. Consistent tool surface; callers don't need to know which endpoints the server honors it on.

2. **Server-side effect is narrow** — Only `opencode_create_session` (POST /session) is documented to process `directory`. All other session endpoints receive it and silently ignore it — this is intentional and harmless. Tool descriptions for those tools say so explicitly.

3. **resolveDirectory() ends at undefined** — NOT process.cwd(). Fallback chain: per-tool param → OPENCODE_DEFAULT_PROJECT → undefined. Sending process.cwd() unconditionally would silently override OpenCode's own session-level directory tracking.

4. **Read OPENCODE_DEFAULT_PROJECT inside the function body** — Not at module scope. Required by INFRA-03 so env changes take effect without restarting Claude Code.

5. **Export resolveDirectory()** — Phase 6 (auto-start) reuses it for child-process cwd resolution.
</locked_decisions>

<anti_patterns>
DO NOT do any of these:
- Read OPENCODE_DEFAULT_PROJECT at module scope (e.g. `const DEFAULT_DIR = process.env.OPENCODE_DEFAULT_PROJECT`). Violates INFRA-03.
- Default to `process.cwd()` in the helper. Violates locked decision #2.
- Skip `resolveDirectory()` for any of the 18 tools. Violates locked decision #1.
- Pass `query: { directory: undefined }` to the SDK (the conditional `dir ? { directory: dir } : undefined` is the established pattern that prevents serialization of undefined values).
- Add a separate `src/utils.ts` or new file. The helper goes in `src/index.ts` near the top.
- Change unrelated code (PartSchema validation, AbortController logic, error handling, CMD-01 model: z.string() schema, etc.).
- Modify the description on `opencode_create_session`'s `directory` field — it already has a tool-specific description. Leave it unchanged.
</anti_patterns>
