## v2.0 Requirements — Final Status

### Session Management

- [x] **SESSION-01**: Claude Code can list all active sessions (`session.list` → GET /session) — **Validated Phase 3**
- [x] **SESSION-02**: Claude Code can fetch a single session by ID (`session.get` → GET /session/:id) — **Validated Phase 3**
- [x] **SESSION-03**: Claude Code can check global session status (`session.status` → GET /session/status) — **Validated Phase 3** *(Note: no sessionId param — global endpoint)*
- [x] **SESSION-04**: Claude Code can retrieve a session's message history with optional limit/pagination (`session.messages` → GET /session/:id/message) — **Validated Phase 3**
- [x] **SESSION-05**: Claude Code can fetch a single message by ID within a session (`session.message` → GET /session/:id/message/:id) — **Validated Phase 3** *(Note: MCP arg `messageId` maps to SDK path param `messageID`)*
- [x] **SESSION-06**: Claude Code can delete a session (`session.delete` → DELETE /session/:id) — **Validated Phase 3**
- [x] **SESSION-07**: Claude Code can rename a session (`session.rename` → PATCH /session/:id via `client.session.update`) — **Validated Phase 3**
- [x] **SESSION-08**: Claude Code can list child sessions of a forked session (`session.children` → GET /session/:id/children) — **Validated Phase 3**
- [x] **SESSION-09**: Claude Code can unrevert a session to undo a revert (`session.unrevert` → POST /session/:id/unrevert) — **Validated Phase 3**

### Run Options

- [x] **RUN-01**: `opencode_run` accepts `model` override (`providerID` + `modelID` pair — both required together, reject if only one is provided) — **Validated Phase 4**
- [x] **RUN-02**: `opencode_run` accepts `agent` to select a specific agent type for a single prompt — **Validated Phase 4**
- [x] **RUN-03**: `opencode_run` accepts `system` to inject a custom system prompt override for a single prompt — **Validated Phase 4**
- [x] **RUN-04**: New `opencode_prompt_async` tool sends a prompt and returns immediately (204) — uses POST /session/:id/prompt_async — **Validated Phase 4**

### Structured Response Surfacing

- [x] **SURF-01**: `opencode_get_diff` surfaces `patch` as a top-level string field in its response — **Validated Phase 4**
- [x] **SURF-02**: `opencode_run` returns parts as a structured typed array — each part tagged by kind via Zod discriminatedUnion schemas for all 12 Part types — **Validated Phase 4** *(Note: discriminator strings verified from @opencode-ai/sdk types before writing schemas)*

### Infrastructure

- [x] **INFRA-01**: `opencode_run` and `opencode_prompt_async` use AbortController + fetch `signal` instead of `Promise.race` — **Validated Phase 4**
- [x] **INFRA-02**: `prefect init` CLI writes a correctly-configured `.mcp.json` into the current working directory — **Validated Phase 4**

### Session Command (added during Phase 4)

- [x] **CMD-01**: `opencode_session_command` tool executes slash commands inside a session via POST /session/:id/command — **Validated Phase 4**

## Deferred to v3.0

- `opencode_run` tools override (enable/disable per prompt)
- `opencode_run` FilePartInput (file attachments as context)
- `opencode_run` messageID (resume from specific message)
- `opencode_run` AgentPartInput / SubtaskPartInput
- `opencode_create_session` parentID (session hierarchies)
- Install script: `npm install -g prefect-mcp` (requires npm publish)
- session.summarize, session.todo, session.init, session.shell, session.share/unshare
- GET /find/symbol, GET /vcs, GET /file/status, GET+POST /mcp
- GET /experimental/tool/ids, GET /experimental/tool, GET /agent, GET /provider

## Out of Scope

- npm packaging / shareable library — personal use tool (npm publish deferred to v3)
- Permission loop (SSE + concurrent HTTP) — OpenCode auto-approves; git is the safety net
- Multi-user or team config — single machine, single developer
- `noReply` body field on opencode_run synchronous endpoint — superseded by `prompt_async`
