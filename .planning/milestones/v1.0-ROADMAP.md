# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-26
**Phases:** 1–2
**Total Plans:** 5

## Overview

Build a TypeScript MCP server that wraps OpenCode's HTTP API as Claude Code tools, then wire it into the Claude Code workflow with documentation and an end-to-end validation task. Phase 1 delivers the working server; Phase 2 makes it usable day-to-day.

## Milestone Summary

**Key Decisions:**
- Auto-approve OpenCode permissions — git is the safety net; permission loop (SSE + blocking HTTP on same session) adds complexity without proportional value
- OPENCODE_URL env var defaults to http://localhost:4096 — easy override without code changes
- opencode_run blocks via Promise.race with PREFECT_TIMEOUT_MS — OpenCode /message is a long-lived HTTP call; 120s default timeout with PREFECT_TIMEOUT_MS override
- StdioServerTransport + .mcp.json project-scope — Claude Code spawns MCP servers as stdio subprocesses; .mcp.json ensures all clones get it without `claude mcp add`
- Permission response enum is `once|always|reject` (not `allow/deny/allow_always` as REQUIREMENTS.md originally stated) — confirmed from @opencode-ai/sdk types

**Issues Resolved:**
- SDK method names differed from RESEARCH.md assumptions (actual: short verbs `create`, `prompt`, `abort`; not `session{Verb}` variants)
- `opencode_approve_permission` is on top-level client, not `client.session`
- tsconfig needed `"types": ["node"]` to resolve `process` global
- REQUIREMENTS.md CORE-04 had wrong permission enum — corrected in implementation and CLAUDE.md

**Issues Deferred:**
- AbortController on fetch calls — PREFECT_TIMEOUT_MS via Promise.race is sufficient for v1; proper cancellation deferred to v2
- Human-needed verification items (MCP tool discovery, E2E loop) — validated live (commit e295cf5) but not automatically verifiable

**Technical Debt:**
- opencode_run only sends TextPartInput — file attachments, agent parts, subtask parts deferred to v3
- 9 SessionPromptData options unexposed — model override, agent selection, noReply, system prompt, tools, messageID all deferred to v2/v3
