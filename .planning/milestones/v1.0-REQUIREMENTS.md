## v1 Requirements — Final Status

### MCP Server Core

- [x] **CORE-01**: User can create a new OpenCode session via `opencode_create_session` (wraps POST /session) — **Validated Phase 1**
- [x] **CORE-02**: User can send a prompt and receive a result via `opencode_run` (wraps POST /session/{id}/message; blocks via Promise.race with PREFECT_TIMEOUT_MS) — **Validated Phase 1**
- [x] **CORE-03**: User can retrieve the file diff for a session via `opencode_get_diff` (wraps GET /session/{id}/diff, optional messageID param) — **Validated Phase 1**
- [x] **CORE-04**: User can respond to an OpenCode permission request via `opencode_approve_permission` (wraps POST /session/{id}/permissions/{permId}, supports `once|always|reject`) — **Validated Phase 1** *(Note: REQUIREMENTS.md originally specified `allow/deny/allow_always` — corrected to match actual API enum)*
- [x] **CORE-05**: User can fork a session at a given message via `opencode_fork` (wraps POST /session/{id}/fork — escape hatch for corrupted sessions) — **Validated Phase 1**
- [x] **CORE-06**: User can revert a session to a prior message via `opencode_revert` (wraps POST /session/{id}/revert) — **Validated Phase 1**
- [x] **CORE-07**: User can abort a running session via `opencode_abort` (wraps POST /session/{id}/abort) — **Validated Phase 1**
- [x] **CORE-08**: OpenCode base URL is read from `OPENCODE_URL` env var, defaulting to `http://localhost:4096` — **Validated Phase 1**

### Wiring & Workflow

- [x] **WIRE-01**: `.mcp.json` at project root registers the MCP server as a Node.js stdio subprocess so Claude Code discovers the tools automatically *(Note: REQUIREMENTS.md said `.claude/settings.json` — research determined `.mcp.json` is the correct mechanism)* — **Validated Phase 2**
- [x] **WIRE-02**: `CLAUDE.md` documents the review/correct loop pattern — create session → run prompt → get diff → run tests → correct or advance — **Validated Phase 2**
- [x] **WIRE-03**: `README.md` covers full setup: install deps, configure and run `opencode serve --port 4096` headless, point Claude Code at the MCP server — **Validated Phase 2**
- [x] **WIRE-04**: `examples/test-task.md` provides a scoped real prompt to validate the full loop end-to-end — **Validated Phase 2** *(commit e295cf5 confirms live run)*

## v2 (Deferred)

- npm packaging / shareable library — personal use for now
- OpenCode config template (`~/.config/opencode/config.json`) with Qwen endpoint wired

## Out of Scope

- SSE-based permission loop — auto-approve in OpenCode config eliminates the need; git is the safety net
- npm publish pipeline — personal tool, not a package
- Multi-user or team configuration — single machine, single developer
